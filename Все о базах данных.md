# Общая информация о базах данных
Тут будет описана общая информация о базах данных, что такое СУБД, что такое реляционная и нереляционная база данных

## Что такое база данных
База данных это некоторое хранилище в котором хранятся данные. Базу данных можно сравнить с библиотекой, в которой может быть безумное количество информации, и ко всей этой информации нужно иметь доступ.
Чтобы грамотно искать данные нужно иметь какое-то средство которое это будет делать. Оно называется **СУБД (система управления базой данных)**.
Есть много СУБД, например, PostgreSQL, Sqlite, Mysql и тд

## Реляционные и нереляционные бд

Все базы данных можно разделить на реляционные и нереляционные.

### Реляционные 
Базы данных такого вида представляют собой таблицы, как например exel. То есть в таких базах данных данные размещаются в таблице со строгой схемой.

| id  | title | description                                        |
| --- | ----- | -------------------------------------------------- |
| 0   | Cook  | This post contains different recipe                |
| 1   | Cars  | This post contains different information about car |
Тут id, title, description являются полями, 0 и далее - записями, а отдельно взятый кусок информации это - ячейка
#### Примеры реляционных СУБД
1. PostgeSQL
2. Mysql
3. oracle database 
и другие

### Нереляционные
Нереляционные бд или NoSQL. Эти базы данных используют различные схемы хранения данных, но не табличную схему.

#### Виды схем в NoSql database
1. Ключ-значение
2. Файловая схема хранения
3. Графы

Такие базы данных используются для специфических задач.

## Типы данных в бд

### SQlite
SQlite поддерживает только 5 типов данных
1. INTEGER - целое число
2. REAL - дробное число
3. TEXT - строка
4. BLOB - двоичные значения
5. NULL - значение, означающее отсутствие информации

# Связи между таблицами
В этом блоке будет рассказываться о связях между таблицами

## Что такое связь в БД и зачем оно вообще надо

Связи в базах данных позволяют таблицам общаться между собой. Это происходит через связывание таблиц с помощью первычных ключей (PK) и внешних ключей (FK).
Например, у нас есть две таблицы: Студент и Курс. У студента есть id (PK) и у курса есть id (PK), и мы можем связать студента и курс с помощью FK. В таблице "Студент" создадим еще одно поле "курс". Там у нас будет указан id курса к которому принадлежит студент.
